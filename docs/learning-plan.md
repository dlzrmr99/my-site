# 个人网站学习计划

创建日期：2026-04-25

## 学习目标

用建立个人网站作为主线，从 0 基础逐步学习前端开发。每个阶段都要产出一个真实可见的网站改进，而不是只看理论。

最终目标：

- 能看懂并修改基本 HTML 结构。
- 能用 CSS 控制布局、颜色、字体、间距和响应式效果。
- 能用少量 JavaScript 做简单交互。
- 能理解文件结构、浏览器预览、Git 记录和基础部署。
- 拥有一个可持续丰富的个人网站。

## 学习脉络

| 阶段 | 主题 | 你要理解的内容 | 网站产出 |
| --- | --- | --- | --- |
| 1 | 认识网页 | HTML 是什么、标签、页面结构、浏览器如何显示网页 | 一个清晰的个人首页骨架 |
| 2 | 内容组织 | 标题、段落、列表、链接、图片、语义化标签 | 首页包含个人简介、兴趣和链接 |
| 3 | 基础样式 | CSS 选择器、颜色、字体、间距、边框 | 页面从默认样式变成有设计感 |
| 4 | 布局 | 盒模型、Flexbox、页面区块、导航栏 | 首页有顶部导航和分区布局 |
| 5 | 响应式 | 手机和电脑屏幕差异、媒体查询 | 页面在手机和桌面都可读 |
| 6 | 作品区 | 卡片、项目信息、外链、图片展示 | 新增作品或学习项目区域 |
| 7 | JavaScript 入门 | 变量、事件、DOM、按钮交互 | 加入主题切换或简单动态内容 |
| 8 | 表单和联系 | 表单元素、基本校验、可访问性 | 新增联系区域或留言表单原型 |
| 9 | Git 基础 | 修改记录、提交、分支、回退概念 | 每次学习都有清楚的版本记录 |
| 10 | 部署上线 | 静态网站、GitHub Pages 或其他托管方式 | 网站获得公开访问地址 |
| 11 | 内容打磨 | 文案、视觉层级、图片、作品介绍 | 网站更像真实个人主页 |
| 12 | 维护迭代 | 重构、整理文件、长期学习记录 | 形成可继续扩展的个人网站 |

## 建议节奏

如果每周学习 2 到 3 次，每次 45 到 90 分钟，可以按 6 到 8 周推进：

| 周次 | 学习重点 | 主要任务 |
| --- | --- | --- |
| 第 1 周 | HTML 基础 | 改造 `index.html`，建立个人首页内容结构 |
| 第 2 周 | CSS 基础 | 新增样式文件，让页面具备基础视觉设计 |
| 第 3 周 | 布局 | 加入导航栏、分区、卡片和更稳定的页面结构 |
| 第 4 周 | 响应式 | 适配手机屏幕，检查文字和布局是否溢出 |
| 第 5 周 | 作品内容 | 添加项目、学习记录或个人经历区块 |
| 第 6 周 | JavaScript | 做一个小交互，例如深浅主题切换 |
| 第 7 周 | 部署 | 学习 Git 和静态网站部署，把网站发布出去 |
| 第 8 周 | 打磨 | 优化文案、配色、细节、可访问性和代码结构 |

## 每次学习模板

每次开始前先确认：

- 本次目标是什么？
- 会涉及哪个文件？
- 需要先学哪个概念？
- 完成后页面应该有什么变化？

每次结束后记录：

- 改了什么？
- 学到了什么？
- 哪些地方还不理解？
- 下一步做什么？

## 第一阶段详细任务

### 第 1 课：认识 `index.html`

目标：

- 理解 `<!doctype html>`、`html`、`head`、`body` 的作用。
- 修改网页标题。
- 把页面内容整理成个人网站首页的基础结构。

建议产出：

- 一个主标题。
- 一段个人介绍。
- 一个“我正在学习什么”的列表。
- 一个“下一步计划”的小区域。

### 第 2 课：常用 HTML 标签

目标：

- 学会使用 `header`、`main`、`section`、`footer`。
- 学会使用 `h1` 到 `h3`、`p`、`ul`、`li`、`a`。

建议产出：

- 首页分成“关于我”“兴趣”“学习计划”“联系方式”几个区域。

### 第 3 课：开始写 CSS

目标：

- 新建 `styles.css`。
- 在 HTML 中通过 `link` 引入 CSS。
- 学会控制字体、背景色、文字颜色、宽度和间距。

建议产出：

- 页面不再是浏览器默认样式。
- 内容区域有统一宽度和舒适间距。

## 学习原则

- 先让页面能运行，再追求漂亮。
- 每次只引入少量新概念。
- 遇到不懂的地方先记录，不急着全部掌握。
- 网站内容要优先表达真实的你，再考虑复杂技术。
- 代码能看懂比代码炫技更重要。
